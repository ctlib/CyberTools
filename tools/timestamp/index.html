How it works

Input parsing: pure integers < 10¹⁰ are treated as Unix seconds; larger integers as Unix milliseconds. String inputs are passed to new Date() which handles ISO 8601, RFC 2822, and most locale-specific formats.

Timezone conversion uses the browser's built-in Intl.DateTimeFormat API with IANA timezone names. This requires a modern browser and has no external dependencies.

The relative time is computed from the current browser clock and updates on each conversion.
